## Items Not Blocking v1.0.0 (justification)

1. **CLA/DCO bot** (S5): The `CONTRIBUTING.md` contains DCO instructions. Automated enforcement is a nice-to-have, not a safety concern. Most major OSS projects add this incrementally.

2. **Backward-compat schema fixtures** (S4): These are needed before schema_version 2 is released, not before v1. There is no previous version to be backward-compatible with.

3. **Per-package LICENSE files** (S6): Root-level `LICENSE-MIT` and `LICENSE-AGPL-3.0` cover the entire repo. Per-package copies improve clarity for packaged distributions but are not legally required given the SPDX headers on every file.

4. **CLI filter/import** (Checklist S6): These were explicitly deferred in the P2 task brief and documented in the release checklist. They are v1.1 features.

5. **Adversarial RLS tests** (S3): The RLS SQL is defined, FORCE is set, and the middleware correctly sets the session variable. The risk is "defined but unverified" -- a post-v1.0.0 priority but not a safety blocker for the OSS release (which is Mode 0, air-gapped, no backend).

6. **Playwright in CI** (S1): Tests exist and pass locally. CI integration requires full-stack orchestration. Not a blocker for the OSS release.
